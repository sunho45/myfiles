Contents for Linux

1) Basic Commands

  1. mkdir <dir>: 디렉터리 생성 — 예) mkdir directory
  2. rmdir <dir>: 비어있는 디렉터리 삭제
  3. rm <file>: 파일 삭제 — 예) rm file.txt
  4. cd <path>: 디렉터리 이동 — 예) cd /home/sunho
  5. cp <src> <dst>: 파일 복사 — 예) cp file.txt file2.txt
  6. mv <src> <dst>: 파일/디렉터리 이동 또는 이름 변경 — 예) mv ee.txt my.txt, mv file.txt /home/sunho/Documents/
  7. chmod: 권한 변경 — 예) chmod u=rwx,g=rwx,o=rwx ee.txt
    숫자 표기: 읽기(4) + 쓰기(2) + 실행(1) → 예) 7(=4+2+1), 6(=4+2), 5(=4+1)
  8. touch <file>: 빈 파일 생성 또는 타임스탬프 갱신
  9. 보기/편집:
    • cat <file>: 파일 내용 보기
    • nano: 간단한 터미널 편집기
    • vi / vim: 고급 편집기 (Vim = Vi Improved)
  10. 목록 보기:
    • ls / ls -l: 파일/디렉터리 목록(+권한/소유자 등)

2) 리눅스 기본 구조

프로그램 ⇄ Shell ⇄ Kernel ⇄ 하드웨어

대표 디렉터리

  • /dev: 장치 파일 (하드웨어 인터페이스)
  • /mnt: 외부 장치 임시 마운트 지점
  • /usr: 사용자 프로그램/라이브러리 등
  • /: 루트(최상위) 디렉터리
  • /boot부트 관련 디렉터리
  • /etc변수,캐시파일이 들어있는 디렉터리
  • /bin일반사용자나 시스템이 사용하는 명령어 모음
  • /home일반사용자의 홈
  • /root시스템 관리자의 홈
  • /sbin루트나 시스템관리자가 사용하는 명령어 모음

3) dpkg(데비안 계열) 명령 표

명령어 설명
sudo dpkg -i package.deb .deb 파일 설치
sudo dpkg -r package_name 설치된 패키지 제거(remove)
sudo dpkg -P package_name 완전 제거(purge, 설정파일 포함)
dpkg -l 설치된 모든 패키지 목록
dpkg -L package_name 해당 패키지가 설치한 파일 목록
dpkg -S /path/file 특정 파일이 어느 패키지에 속하는지 확인
dpkg -s package_name 패키지 상태(status) 확인

네트워크 관련 지식

용어 설명
DNS 도메인 이름을 IP 주소로 바꾸는 시스템/프로토콜
Gateway 외부 네트워크로 나가는 출입구(일반적으로 라우터)
MTU 한 번에 전송 가능한 최대 패킷 크기
Latency (RTT) 왕복 지연 시간. 요청→응답에 걸리는 시간

네트워크 관련 명령어

  • traceroute: 목적지까지의 경로를 hop 단위로 추적 — 예) traceroute google.com
  • ping: 대상 호스트와의 연결/지연 확인 — 예) ping 8.8.8.8
  • ss: 소켓 상태 확인 — 예) ss -tuln
  • ip: 주소/링크/라우팅 관리 — 예) ip addr, ip route
  • dig/nslookup: DNS 질의 — 예) dig A google.com
  • tcpdump: 패킷 캡처 — 예) sudo tcpdump -i any port 80

※ 일부 명령은 패키지 설치가 필요할 수 있습니다(예: sudo apt install traceroute).

Network Cast Types — Quick Sheet
Last updated: 2025-11-03

유니캐스트 · 브로드캐스트 · 멀티캐스트 한눈 정리

“누구에게 보내는가” 기준의 전송 방식 요약. 예시와 주소 범위, 사용 프로토콜, 장단점을 함께 묶었습니다.

Unicast
1:1 통신 · 일반 호스트 ↔ 호스트
  • 대상: 특정 단말 1대
  • 예: HTTP, SSH, FTP
  • IP: 일반 호스트 주소
  • MAC: 대상 호스트 MAC
Broadcast
1:모두 · 동일 서브넷 전체
  • 대상: 같은 네트워크의 전 장비
  • 예: ARP Request, DHCP Discover
  • IP: 255.255.255.255
  • MAC: FF:FF:FF:FF:FF:FF
Multicast
1:그룹 · 가입자만 수신
  • 대상: 특정 그룹(N개)
  • 예: IPTV, 화상회의, 라우팅 프로토콜(OSPF)
  • IP: 224.0.0.0 ~ 239.255.255.255
  • MAC: 01:00:5E:xx:xx:xx
